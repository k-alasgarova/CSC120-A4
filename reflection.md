-Initially, encaptulation was overwhelming as I thought of all the work it entailed. Instead of a one large program, there multiple classes that have to control certain information, and I have to decide what goes where. In this assignemnt, the Engine manages fuel, the Car passangers, and the Train - all together. While overwhelming at first, it is actually a better better for a more organized program, but there is a lot of planning involved too. 

At the same time, encapsulation was useful as a lot of mistakes can be prevented when certain class has its own data. For example, Passangers couldn't directly modify a passanger's list but it uses methods like boardCar() and getOffCar().

- What worked well was understanding first the relationship between the classes and then starting with Engine and Cars first before the Train - this was it was easier to see how Train and Passengers interact. I had run into some problems with interface errors, mainly when method names or return types don't exactly match the required interfaces. These small differences confused me at first, but once again, remined that Java is sensitive in this sense, so I have to be precise. 

For someone taking this course, I would advice to implement one class at a time andcheck the method names with interface requirements - a small mishap can prevent a program from compiling.
